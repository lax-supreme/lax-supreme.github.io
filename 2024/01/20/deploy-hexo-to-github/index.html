如何将Hexo博客部署到GitHub Pages

在完成Hexo博客的本地搭建后，我们需要将博客部署到GitHub Pages上，让其他人能够访问我们的博客。本文将详细介绍部署的完整步骤。

准备工作

  1. 确保已经有GitHub账号
  2. 确保本地已经安装了Git
  3. 确保Hexo博客能在本地正常运行

步骤一：创建GitHub仓库

  1. 登录GitHub账号
  2. 创建新的仓库，仓库名必须是：你的用户名.github.io
    • 例如你的GitHub用户名是lax-supreme，则仓库名应该是lax-supreme.github.io
  3. 将仓库设为公开（Public）

步骤六：访问博客

等待几分钟后，就可以通过 https://你的用户名.github.io 访问你的博客了。

常见问题

1. 部署失败

  • 检查仓库名是否正确
  • 检查deploy配置是否正确
  • 检查是否有权限访问仓库

2. 页面404

  • 确保仓库名完全正确
  • 等待几分钟，GitHub Pages需要一定时间来部署

3. 样式丢失

  • 检查_config.yml中的网站URL配置
  • 确保主题文件完整

步骤七：更新博客内容

每次要更新博客内容时，按以下步骤操作：

  1. 创建新文章

  2. 编辑文章内容（文章位于 source/_posts/ 目录下）

  3. 本地预览

  4. 确认无误后，部署更新

  5. 部署完成后，等待几分钟即可在网站上看到更新

维护建议

  1. 定期备份源文件
  2. 建议创建新分支存储源文件
  3. 养成良好的提交习惯

总结

按照以上步骤，你应该已经成功将Hexo博客部署到GitHub Pages上了。如果遇到问题，可以查看官方文档或者留言询问。

记住，部署成功后要经常更新内容，保持博客的活力！
